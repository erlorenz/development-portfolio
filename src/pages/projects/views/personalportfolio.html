Overview

Multi page static site to display various projects, including technologies/libraries used and some challenges that were overcome.

Uses modern CSS features, javascript APIs, and custom configuration.

Modern Features/APIs

PROBLEM: Most projects are using a javascript framework, CSS library, and/or libraries for interactivity and structure.
For a static site, modern features in CSS and javascript can do plenty.

SOLUTION: For reuseable/stateful components the newly released Lit-Element from Polymer is a simple web components wrapper.
CSS grid, CSS variables, scroll-behavior, and intersection observer allows layouts and interactions that previously required libraries.

Configuration and Tooling

PROBLEM: Most configuration tools and bundlers are aimed at Single Page Applications.
For a multi-page site there is a lot more configuration needed to bundle, code split, etc. to achieve the same result.

SOLUTION: Modified the Webpack/TS Boilerplate for use with multiple entry points and HTML pages.
Gatsby.js' configuration guide was used as a resource for optimizations (preloading/prefetching, code-splitting/caching) to improve user experience.
