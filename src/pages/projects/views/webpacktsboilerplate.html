Overview

A feature-complete starter setup for either a single or multi page application.

Out of the box it is configured to lint, format, and transpile Javascript/Typescript while also loading, code-splitting, and minifying all assets.

Magical Configuration

PROBLEM: Due to complexity of configuration most projects are using either a non-configurable environment like Create-React-App or super highly configured boilerplates with too much abstraction.
To eject or modify these setups you begin at a confusing point without any understanding of the configuration.

SOLUTION: This project was started from scratch using the Webpack documentation as well as multiple guides.
Then step-by-step each loader/optimizer/plugin was added when necessary.
Moving forward this serves as an excellent base with only very framework or library-specific tools to add.

Editor Tooling

PROBLEM: Due to the sheer amount of possible ways to lint, format, and transpile code there are very few best practices.
Even when there are good working methods, new features and changes make it difficult to keep track with what is outdated and not.

SOLUTION: Previously the preferred setup was TypeScript + TSLint or Babel + ESLint which were always changing and required different configuration on a per-project basis.
With TypeScript moving to ESLint only, this allows a TypeScript + ESLint + Prettier + VSCode setup that can be used in every single project with minimal modification.
